Development Journey

How We Built This

Challenges overcome and innovations implemented in creating this intelligent shopping assistant

Phase 1: Foundation
LLM Integration & Architecture
Set up FastAPI backend with LLM integration, designed conversation flow, and built core chat infrastructure. Challenge: CPU-limited local LLM (~6 mins/response) Solution: Switched to DeepSeek & Gemini APIs (reduced to seconds)
Phase 2: NLP & Intent Classification
Smart Shopping Intent Recognition
Implemented intent classification system to distinguish between product search, cart operations, and recipe planning using LLM function calling. Challenge: Context loop confusion which causes LLM to repeat outputs Solution: Hardcoded logic to prevent repeated llm calls. Which led to better performance and fewer tokens.
Phase 3: Image Processing
OCR & Shopping List Extraction
Integrated image recognition and OCR to extract shopping items from handwritten photos using Gemini Vision API with rapid processing. Challenge: Handwriting accuracy and variant recognition Solution: Multi-API approach with fuzzy matching fallback
Phase 4: Database & Inventory
Product Database Integration
Connected to product inventory database with real-time stock checking and pricing information for accurate suggestions. Challenge: Query performance at scale Solution: Intelligent caching and indexing strategies
Phase 5: Frontend & UX
Interactive Chat Interface
Built responsive chat UI with streaming responses, real-time cart updates, and modern 2026-style design aesthetics. Challenge: Real-time synchronization between components Solution: Event-driven architecture with localStorage cache
Phase 6: Testing & Optimization
Quality Assurance & Performance
Comprehensive testing, performance optimization, and user feedback integration for production readiness and reliability. Challenge: Edge case handling and error resilience Solution: Robust error handling and graceful degradation
